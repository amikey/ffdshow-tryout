What is the difference between generic builds, SSE builds and generic builds (ICL 9.1)?

  • Generic builds work on Pentium-MMX/AMD K6 or later.
  • SSE builds work on Pentium III/Athlon XP or later.
  • SSE2 builds work on Pentium 4/Athlon 64 or later.
  • Generic builds (ICL 9.1) work on Pentium-MMX/AMD K6 or later.

ffdshow's internal decoders and resize filter detect the CPU's supported instruction sets like SSE/SSE2/3DNow! automatically and achieve optimal performance even if the builds are "generic". Performance-critical parts are programmed in "hand-optimized" assembler code. So there is almost no room for compiler optimization.

libavcodec - DivX5 720p CinderellaMan_HD.avi
buildsresults (sec)
ffdshow_rev752_20070108_xxl.exe 42
ffdshow_rev752_20070108_sse_xxl.exe 41
ffdshow_rev756_20070109_clsid_icl9.exe 42

The difference between generic and SSE/ICL 9.1 builds is negligible when using ffdshow just for basic decoding.

CinderellaMan_HD.avi + xsharpen
buildsresults (sec)
ffdshow_rev752_20070108_xxl.exe 148
ffdshow_rev752_20070108_sse_xxl.exe 90
ffdshow_rev756_20070109_clsid_icl9.exe 102

SSE builds are faster than generic builds if you use image filters such as xsharpen.

CinderellaMan_HD.avi + denoise3d
buildsresults(sec)
ffdshow_rev752_20070108_xxl.exe 131
ffdshow_rev752_20070108_sse_xxl.exe 130
ffdshow_rev756_20070109_clsid_icl9.exe 115

ICL 9.1 generates faster code with or without SSE.
For example, when you use denoise3d, ICL 9.1 builds are faster than generic or SSE builds.

Both SSE/SSE2 and ICL 9.1 builds can have an impact on performance when using one of ffdshow's filters like blur, sharpen, denoise3d etc. Non-libavcodec decoders like libdts and libfaad can show a noticeable boost too.
The improvements over the generic builds differ from system to system though. So it is recommended to test the provided builds on your own system.
For benchmarking purposes, you will need Haali's timeCodec.exe [96KB]. For timeCodec.exe to work, Haali Media Splitter is needed.

Generic builds may have better stability compared to those optimized ones. If you are new to ffdshow and only interested in basic decoding of your audio/video files, we recommend you to try the generic builds first. If you want to further improve performance and intend to use some of ffdshow's filters, then you should give the SSE/SSE2/ICL 9.1 builds a try.

Is ffdshow multithreaded?

  • Resize is fully multithreaded. It is faster on a dual-core CPU. No configuration is needed. It is not effective on Pentium 4 HT CPUs.
  • With 'Queue & misc'-'Queue output samples' checked, acceleration using multithreading is enabled. You may see some benefit even on a single-core CPU.
  • libavcodec's MPEG1/2 decoders are multithreaded. It becomes effective when setting 'Decoder options'-'Number of decoding threads' to the number of supported cores. The setting has no effect on Pentium 4 HT CPUs.
  • libavcodec's MPEG4/MPEG2/MPEG1 encoders are multithreaded.
