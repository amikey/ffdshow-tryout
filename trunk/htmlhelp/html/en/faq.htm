What's the difference between generic builds, SSE builds and generic builds(ICL 9.1)?

  • generic builds work on Pentium-mmx/AMD K6 or later.
  • SSE builds work on Pentium3/Athlon Xp or later.
  • SSE2 builds work on Pentium4/Athlon64 or later.
  • generic builds(ICL 9.1) work on Pentium-mmx/AMD K6 or later.

Cardinal decoders and resize filter detect SSE/SSE2 automatically and achieve optimal performance even if the builds are "generic".
They are written in assembler language and there are almost no room for compiler optimization.

libavcodec-DivX 5 720p CinderellaMan_HD.avi
buildsresults(sec)
ffdshow_rev752_20070108_xxl.exe42
ffdshow_rev752_20070108_sse_xxl.exe41
ffdshow_rev756_20070109_clsid_icl9.exe42

SSE builds are faster than generic bulds if you use image filters such as xsharpen.

CinderellaMan_HD.avi + xsharpen
buildsresults(sec)
ffdshow_rev752_20070108_xxl.exe148
ffdshow_rev752_20070108_sse_xxl.exe90
ffdshow_rev756_20070109_clsid_icl9.exe102

ICL 9.1 generates faster code with or without SSE.
For example when you use denoise 3d, ICL 9.1 builds are faster than generic builds or SSE builds.

CinderellaMan_HD.avi + denoise 3d
buildsresults(sec)
ffdshow_rev752_20070108_xxl.exe131
ffdshow_rev752_20070108_sse_xxl.exe130
ffdshow_rev756_20070109_clsid_icl9.exe115

Both SSE and ICL 9.1 builds have good advantage in using certain image filters. Each has likes and dislikes.
It is interesting to reserch that issue. You'll need Haali's timeCodec.exe.

Generic builds may have better stability compared to those optimized builds.
If you are first to use ffdshow, we recommend you to try generic builds.
If your CPU can't catch up when you use image filters, you may try SSE/ICL 9.1 builds.

Is ffdshow multithreaded?

  • Resize is fully multithreaded. It is faster on dual-core CPU. No configuration is needed. It is not effective on Pentium4 HT.
  • With 'Queue & misc'-'Queue output samples' checked, acceleration using multithreading is enabled. You may see some benefit on single CPU.
  • libavcodec mpeg1/2 decoders are multithreaded. By setting 'Decoder options'-'Number of decoding threads', it is accelerated. It is not effective on Pentium4 HT.
  • libavcodec MPEG4/MPEG2/MPEG1 encoders are multithreaded.
